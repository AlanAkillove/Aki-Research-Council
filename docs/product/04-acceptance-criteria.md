# ARC 验收清单（Acceptance Criteria）

> **用途**：阶段完成的可检查定义。未勾选不得宣称该阶段「完成」。  
> **版本**：v0.3  
> **关联**：[`01-prd-summary.md`](./01-prd-summary.md) · [`02-architecture.md`](./02-architecture.md) · [`03-tech-spec.md`](./03-tech-spec.md)

图例：`[ ]` 未完成 · `[x]` 已完成 · `N/A` 本阶段不适用

---

## 0. 全局质量门禁（所有阶段共用）

- [ ] 重要判断均有来源（Evidence ID 或可点击链接），无裸断言
- [ ] Claim 类型未把 `inference` / `hypothesis` 标成 `fact`
- [ ] 新颖性表述落在 Tech Spec 五档枚举内，无「首创/全新」绝对措辞
- [ ] 同一逻辑论文（含多版本）不在同一日作为两条「新论文」推荐
- [ ] 日报头版 ≤3；精读 ≤4；新 Idea ≤1；动作 ≤3
- [ ] 允许并正确渲染「今日无足以改变判断的新证据」
- [ ] 每次 run 写入成本/模型/Prompt/git 复现日志
- [ ] 采集或分析失败时能降级出 `partial` 报告，不静默空洞成功
- [ ] 无私有全文或密钥泄漏到公开 Pages

---

## 阶段 P1：可信的研究日报

### P1.1 功能

- [ ] arXiv 按配置类别增量抓取
- [ ] OpenReview / Semantic Scholar / OpenAlex 至少各有一条可用接入路径
- [ ] 元数据归一 + DOI/arXiv/标题指纹去重
- [ ] 版本更新识别为 update 事件（至少记录 version 列表）
- [ ] `research_state` 可配置 ≥1 个项目与开放问题
- [ ] 两阶段筛选：硬过滤 → LLM 多维 JSON 精排
- [ ] 三个频道可开关：`ai_for_math` / `structured_and_reliable_vision` / `combinatorics_and_matroids`
- [ ] 生成每日 Markdown + 自包含 HTML（Jinja2；色板与第一屏结构符合 [`05-ui-design.md`](./05-ui-design.md)）
- [ ] 基础反馈可写入 `feedback.jsonl`（不改历史报告）

### P1.2 验收标准（通过条件）

- [ ] **去重**：连续 3 个工作日，人工抽查无「同稿多版本当新稿」误报
- [ ] **来源**：抽查头版与精读条目，100% 有来源字段
- [ ] **分域**：AI 与数学论文使用不同分析模板字段（缺字段则失败）
- [ ] **相关性**：人工标「值得浏览」的论文，当日 Precision@5 ≥ 0.6（冷启动可先 ≥0.4 并记录基线）
- [ ] **时限**：用户阅读主路径（头版+State Changes+动作）目标 ≤15 分钟

### P1.3 退出标准

P1 全部 P1.2 通过后，才启动 P2 全文证据与多角色评审。

---

## 阶段 P2：证据驱动的虚拟组会

### P2.1 功能

- [ ] 对 ≤5 篇/日进行全文/TeX 证据抽取，生成 Evidence Pack
- [ ] Historian / Skeptic / Liaison 结构化输出并入库
- [ ] Claim Ledger 可追加；写入正式 State 须 `approved_by: chair`
- [ ] 日报含 **Research State Changes** 栏目
- [ ] 能关联到具体 `project_id` / `question_id`
- [ ] 周会 Markdown 纪要可生成

### P2.2 验收标准

- [ ] 抽查精读论文：能指出 ≥1 篇最接近前作（或显式「检索未找到」）
- [ ] 能区分「作者主张」与「实际增量/Agent 推断」
- [ ] 对至少 5 篇样本，正确回答「是否改变已有判断」且用户认可率 ≥ 80%
- [ ] Skeptic 在证据不足时输出「证据不足」，而非强制反对
- [ ] Unsupported Claim Rate（抽查）≤ 10%

### P2.3 退出标准

P2.2 通过后进入 P3。

---

## 阶段 P3：长期记忆与研究决策

### P3.1 功能

- [ ] Idea 生命周期全状态可迁移；rejected 不可物理删除
- [ ] Chair 决议：`READ|TRY|WATCH|ARCHIVE|NO-GO|UPDATE`
- [ ] 周会想法锦标赛（Generation / Skeptic / Feasibility）且每周 ≤1 个进入待验证
- [ ] 可编辑周会 PPTX（python-pptx）+ Markdown 纪要
- [ ] 反馈校准排序；输出月度权重敏感性摘要
- [ ] 70/20/10 探索配比可配置并在附录可审计
- [ ] 月度回顾报告可生成

### P3.2 验收标准

- [ ] 连续两周：已否决方向复发推荐率 ≤ 5%
- [ ] 新重点论文 ≥ 80% 自动关联到开放问题或显式「仅雷达」
- [ ] 每周任务输出 ≤3 且可执行（含具体章节/公式/实验，而非「继续调研」）
- [ ] Idea Survival：进入 candidate 后两周仍被用户认为有价值的比例有基线记录
- [ ] NO-GO 至少产生过有效否决并阻止重复提案（案例可追溯）

### P3.3 退出标准

P3.2 通过后，才考虑 P4 执行器。

---

## 阶段 P4：研究验证助手（可选）

### P4.1 功能

- [ ] 从 Idea 生成最小验证协议（步骤、期望、失败条件）
- [ ] 可选：小规模脚本/计算辅助与结果回写
- [ ] 根据失败更新 hypothesis / rejected
- [ ] **禁止**：自动修改正式论文、自动投稿、擅自大规模训练

### P4.2 验收标准

- [ ] 验证协议均可人工独立执行
- [ ] 结果回写不覆盖原始实验日志（只追加）
- [ ] 任何自动执行都需显式用户确认开关（默认关）

---

## 横切验收：版本差分与多样性

- [ ] 论文从 v1→v2 时生成差异报告（至少摘要/作者/结论强度字段）
- [ ] 当候选前十主题熵过低时，雷达区出现多样性保留结果；头版仍可纯价值排序
- [ ] Missed Paper Audit：用户事后标「重要但未出现」的条目可记录原因标签

---

## 横切验收：评测与成本

- [ ] 离线标注集 ≥ 100 篇（可分阶段累积）
- [ ] 仪表或报告可见：Precision@5、Duplicate Rate、日成本
- [ ] 单日强模型全文分析次数不超过 `fulltext_analyzed` 配置

---

## 发布验收（每次上线）

- [ ] `docs/product/` 四件套与实现无已知冲突；若有冲突已开 issue
- [ ] README 写明如何配置 API 密钥与首次生成日报
- [ ] 主路径 smoke test：ingest → screen → report 在 CI 或本地脚本通过
- [ ] 变更日志记录用户可见行为变化
