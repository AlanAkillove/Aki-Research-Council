# ARC 产品原则（PRD 摘要）

> **产品名**：ARC（Aki Research Council）  
> **一句话**：持续理解研究者当前项目与约束，用证据更新判断，并输出可执行下一步的个人研究委员会。  
> **版本**：v0.3（产品原则）  
> **非目标**：不是 arXiv 摘要器，不是自动完成科研的 AI Scientist。

---

## 1. 为谁、解决什么

| 项 | 内容 |
|----|------|
| 用户 | 个人研究者（数学 / AI·CV / AI for Math） |
| 痛点 | 前沿信息过载；日报好看但不可行动；Idea 重复发散；忘记否决历史；无法判断新证据是否改变旧判断 |
| 成功标准 | 每天 8–15 分钟读完晨报后，能明确「今天做什么 / 不做什么」；周会能收敛到 ≤3 个可执行任务 |

---

## 2. 核心流程（产品逻辑）

普通日报：

```text
新论文 → 摘要 → 推荐列表
```

ARC：

```text
当前研究状态 + 新增证据
→ 判断是否改变
→ 候选假设（可选）
→ 最小验证 / 精读 / NO-GO
→ 写回 Research State
```

**基本处理单元是研究问题，不是论文。**

---

## 3. 不可妥协的原则

1. **问题中心**：无法关联到项目 / 开放问题 / 长期兴趣的内容，只能进雷达，不能占头版。
2. **证据优先于生成**：区分原文事实、作者主张、Agent 分析、推断、未验证想法；禁止把推断写成事实。
3. **长期状态优先于单次 Prompt**：必须记住项目、否决理由、资源约束与历史决议。
4. **明确支持 NO-GO**：否决与推荐同等重要；否决必须可追溯、可复活条件。
5. **行动上限**：头版 ≤3 条；每日精读 ≤3–4 篇；新 Idea ≤1；动作 ≤3。
6. **无变化是合法输出**：允许写「今日没有足以改变现有研究判断的新证据」。
7. **人做最终决策**：系统只给证据、质疑与最小验证协议，不自动训练 / 投稿 / 改正式论文。

---

## 4. 第一版服务范围

三个研究频道：

- `ai_for_math`
- `structured_and_reliable_vision`
- `combinatorics_and_matroids`

内置约束（摘要）：偏好理论与机制；拒大规模预训练；重视死亡条件与可证伪实验；区分信号 / 候选 / 正式项目。

---

## 5. 核心产物

| 节奏 | 产物 | 用户价值 |
|------|------|----------|
| 每日 | Markdown + HTML 晨报 | 状态是否变化 + 至多 3 个动作 |
| 每周 | 组会纪要 + 可编辑 PPT | GO / WATCH / NO-GO + 下周任务 |
| 专题 | 深度备忘录 | 用户指定问题的路线图与最小实验 |
| 长期 | Research State + Claim Ledger + Idea/Decision 历史 | 避免循环发散 |

---

## 6. 明确不做（v1）

- 每日全量多 Agent 自由群聊或昂贵自博弈
- 宣称「首创 / 全新」而不做检索限定表述
- 无证据链的想法直接写入研究方向
- 自动改代码训练、自动投稿
- 绕过付费墙或再分发无授权全文

---

## 7. 价值壁垒（产品检验句）

若去掉漂亮排版与 Agent 数量，系统是否仍能：

1. 长期理解我正在解决什么；
2. 用证据区分新信号与伪趋势；
3. 记住失败与否决；
4. 把前沿变成可证伪任务；
5. 在新证据出现时**明确修改旧判断**。

若不能，则产品失败——无论抓了多少论文。

---

## 8. 阶段目标（产品视角）

| 阶段 | 目标 | 一句话验收 |
|------|------|------------|
| P1 | 可信研究日报 | 前五推荐大多值得浏览；判断有来源；不重复同一版本 |
| P2 | 证据驱动组会 | 能说清增量、反对意见、是否改变旧判断 |
| P3 | 长期记忆与决策 | 少重复否决方向；周会 ≤3 任务；权重可校准 |
| P4 | 验证助手（可选） | 最小实验协议与结果回写；仍不自动提交 |

详见 [`04-acceptance-criteria.md`](./04-acceptance-criteria.md)。
