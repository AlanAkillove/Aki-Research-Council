# ADR 0001 — 初始技术栈与状态写入权威

## 状态

Accepted（实现默认值由 [ADR-0002](./0002-stack-revision.md) 修订）

## 日期

2026-07-18

## 背景

ARC 需要可本地运行的个人研究管线：长期 Research State、证据可追溯、成本可控。需在「过早上分布式基础设施」与「毫无结构的脚本」之间取舍；并明确谁能改正式研究判断。

## 决策（原则层，仍然有效）

1. **存储**：MVP 用 SQLite + JSONL + Git 管理报告/配置。数据显著增长后再评估 PostgreSQL。
2. **编排**：显式状态机，不先引入重型多 Agent 框架。
3. **权威**：仅 **Chair** 可写入正式 Research State；其他角色产出草稿/证据。
4. **模型**：厂商无关的 `ModelProvider`；强模型仅用于少量全文与周会（具体客户端见 ADR-0002）。
5. **发布**：Markdown 为内容事实源；HTML 日常读；周会另有可编辑幻灯片（具体工具见 ADR-0002）。

## 替代方案

| 方案 | 否决原因 |
|------|----------|
| 每日多 Agent 自由群聊 | 成本高、难复现、顺序决策任务未必更好 |
| 一开始上 Postgres + 独立向量数据库 | MVP 过重 |
| 任意角色可写 State | 判断漂移、难审计 |
| 绑定单一厂商 SDK | 锁定风险 |

## 后果

- 实现简单、可审计；Chair 成为瓶颈但是有意为之。
- 以后换存储/框架须新开 ADR，并迁移 State 写入接口。
- 组件级默认选型以 ADR-0002 为准。
